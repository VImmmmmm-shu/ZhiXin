G1 机器人换装（衣服材质替换）和帽子造型（独立控制）的完整过程
一、换装实现（材质替换法）
核心思路
不再使用复制身体网格作为“衣服组”，而是直接在机器人身上替换身体材质，实现颜色变化。这样做无需解压预制体，动画不受影响，且完美贴合体型。
实现步骤
1. 准备材质球
•	创建四种颜色材质：RedCloth、BlueCloth、ArmorMaterial（灰色/金属）、GreenCloth。
•	材质类型可以是 Standard 或 URP/Lit，颜色随意调整。
2. 编写 Wardrobe.cs 脚本
•	添加 public Material[] bodyMaterials 数组（顺序：0红,1蓝,2灰,3绿）。
•	在 Start() 中获取机器人所有 SkinnedMeshRenderer（或 MeshRenderer），并默认设置第一个材质。
•	提供 SwitchTo(int index, bool syncHat) 方法，切换身体材质，并可选择是否同步帽子颜色。
•	提供 SetBodyColor(int index) 直接替换材质。
3. 挂载并配置
•	将 Wardrobe 脚本挂载到机器人根物体（例如 G1 或 g1chan）。
•	在 Inspector 中，Body Materials 数组大小设为 4，依次拖入四种材质。
4. 语音指令调用
•	在 SpeechToTextAndLLM.cs 中，根据识别到的颜色关键词调用 wardrobe.SwitchTo(index, false)（不同步帽子）或复合指令中分别处理衣服和帽子。
二、帽子造型实现（独立挂点 + 材质替换）
核心思路
制作帽子模型 → 绑定到头部骨骼 → 通过更换帽子材质实现颜色变化 → 语音控制戴/摘/换色。
实现步骤
1. 制作帽子模型（使用基本几何体）
•	创建空物体 HatGroup。
•	添加 Cylinder 作为帽檐（Scale: 0.5,0.02,0.5）。
•	添加 Cylinder 作为帽顶（Scale: 0.3,0.12,0.3，Position: 0,0.07,0）。
•	可选添加 Sphere 作为帽顶小球（Scale: 0.08,0.08,0.08，Position: 0,0.15,0）。
2. 绑定到头部骨骼
•	找到机器人头部动画骨骼（如 head_link、head_link_0、mixamorig:Head）。
•	将 HatGroup 拖拽到该骨骼下，成为子物体。
•	调整 HatGroup 的局部位置（例如 Y=0.22, Z=0.08），使帽子戴正。
3. 在 Wardrobe.cs 中添加帽子控制
•	添加 public GameObject hatGroup 和 public Material[] hatMaterials（顺序与衣服材质一致）。
•	实现 SetHatMaterial(Material mat)、SetHatColor(int index)、ShowHat()、HideHat()、RemoveHat()、SetHatColorAndShow(int index)。
•	在 SwitchTo(int index, bool syncHat) 中，当 syncHat 为 true 时调用 SetHatColorAndShow(index)。
4. 配置 Inspector
•	将 HatGroup 拖入 Wardrobe 组件的 Hat Group 字段。
•	设置 Hat Materials 数组大小为 4，依次拖入 RedCloth、BlueCloth、ArmorMaterial、GreenCloth。
5. 语音指令控制帽子
•	在 SpeechToTextAndLLM.cs 中增加 ProcessHatCommand 方法，识别：
o	“取下帽子”、“摘帽子” → wardrobe.RemoveHat()
o	“戴上红色帽子” → wardrobe.SetHatColorAndShow(0)
o	“戴上帽子”（无颜色） → wardrobe.ShowHat()
•	在 StopRecording 中优先调用 ProcessHatCommand，避免与衣服指令冲突。
6. 混搭支持（衣服+帽子不同颜色）
•	在 ProcessClothingCommand 中检测复合指令（同时包含“衣服”和“帽子”）。
•	分别提取衣服颜色和帽子颜色，调用 wardrobe.SwitchTo(clothColor, false) 和 wardrobe.SetHatColorAndShow(hatColor)。
•	示例指令：“穿上红色的衣服并带上蓝色的帽子” → 衣服变红，帽子变蓝。
