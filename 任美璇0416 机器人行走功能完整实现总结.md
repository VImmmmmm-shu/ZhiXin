
机器人行走功能完整实现总结
本总结涵盖从平移移动到真实行走动画的全部过程，包括设置步骤、常见问题及解决方案。
一、前置条件
•	机器人已有NavMeshAgent组件（用于导航移动）。
•	场景已烘焙NavMesh（可行走区域）。
•	机器人已有Animator组件，且已有待机（Idle）动画。

1. 准备行走动画
•	从Mixamo下载FBX for Unity格式的 Walk 动画。
•	导入 Unity，确保Rig类型为Humanoid，Animation标签中勾选Loop Time。
2. 创建/修改 Animator Controller
•	打开机器人使用的 Animator Controller（如 ctrl_Piano）。
•	创建Speed参数：Parameters→+→Float，命名Speed。
•	创建混合树（Blend Tree）：
o	右键 →Create State→From New Blend Tree，命名为Movement。
o	双击进入混合树，设置：
	Blend Type = 1D
	Parameter = Speed
	添加两个 Motion：
	第一个：Idle 动画，Threshold = 0
	第二个：Walk 动画，Threshold = 1
o	返回 Base Layer。
•	设为默认状态：右键 Movement → Set as Layer Default State。
3. 编写 NavAgentAnimator 脚本
csharp
using UnityEngine;
using UnityEngine.AI;

[RequireComponent(typeof(NavMeshAgent), typeof(Animator))]
public class NavAgentAnimator : MonoBehaviour
{
    private NavMeshAgent agent;
    private Animator animator;

    void Start()
    {
        agent = GetComponent<NavMeshAgent>();
        animator = GetComponent<Animator>();
    }

    void Update()
    {
        if (agent != null && animator != null)
        {
            float speed = agent.velocity.magnitude;
            animator.SetFloat("Speed", speed, 0.1f, Time.deltaTime);
        }
    }
}
•	将脚本挂载到机器人上。
5. 测试行走
•	运行游戏，发出移动指令（如“去客厅”）。
•	机器人应移动且播放行走动画，停止时播放待机动画。
